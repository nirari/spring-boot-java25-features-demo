⚡ Java 25 Virtual Threads Demo

This interactive demo showcases Virtual Threads (Project Loom) introduced in Java 25. Virtual threads enable massive concurrency with minimal overhead compared to traditional platform threads.

Key concepts:

  • Virtual Threads: Lightweight threads managed by the JVM. Can spawn 10,000+ concurrently.
  • Platform Threads: Traditional OS threads. Limited to ~1000 concurrent threads due to OS resource constraints.
  • I/O-bound tasks: Tasks that spend most time waiting (DB queries, HTTP calls). Virtual threads excel here.
  • CPU-bound tasks: Tasks that use CPU continuously. Platform threads can be slightly faster.

📚 About Virtual Threads

What are Virtual Threads?

Virtual threads are lightweight threads that are scheduled by the Java Virtual Machine (JVM), not by the OS. Unlike platform threads (1:1 OS thread mapping), virtual threads can be created by the thousands without exhausting system resources.

Why use Virtual Threads?

  • I/O-bound applications (web servers, APIs) can handle massive concurrency
  • Simplify async programming - you can write blocking code without wasting threads
  • Same programming model as traditional threads (no callbacks/Futures needed)
  • Millions of virtual threads can coexist on a few platform threads

When not to use Virtual Threads?

  • CPU-intensive computations (use platform threads or parallel streams)
  • When you need direct OS thread control (affinity, priority)

Performance notes:

  • Virtual thread creation: ~2KB stack vs ~1-2MB for platform threads
  • Context switch: ~100ns (virtual) vs ~1-10µs (platform)
  • Blocking I/O doesn't waste OS resources
